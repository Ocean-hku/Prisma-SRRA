# 开发流程与需求迭代记录 (格式润色版)

## 一、 产品立项与概念定义

- **产品初衷**：当前的心理测试分类过少、边界死板且结果含糊 。希望结合社会学与心理学，打造一个分类细致、具备灵活边界（soft margin）且能提供深刻自我认知的产品 。
- **初期草案**：要求生成正式的产品提案（proposal）与可交互的网页草案 。
- **内容规格**：设定至少50个精准分类问题与40种（后精简为16种）可爱精致的动物/形象标签 。测试结果需以百分比展示主次标签，并提供世界线互动、微实验行动以及多场景的详细诊断 。

## 二、 题库重构与本土化场景设计

- **场景化结构**：摈弃重复空洞的题目，采用“5大中国高频语境”（职场/学校、家庭/代际、关系/亲密、社交媒体、消费/风险）与“四层结构”（平时表现、特定场景、动机状态、自我故事）进行设计 。
- **选项优化**：由于部分题目场景过于极端且选项不匹配，要求结合具体热点事件出题，并取消传统的抽象量表形式 。改为提供5个具体的描述性选项，分别对应 -2 到 2 的评分权重 。

## 三、 核心算法与科学测算原理建立

- **四维基础空间**：建立了由社交(Social)、理性(Rational)、叛逆(Rebellious)、野心(Ambition)组成的四维向量基础空间 。为了迎合年轻群体，后续要求将拗口的“SRRA”重新命名 。
- **距离与权重算法**：引入了“质心(Centroid)”概念，系统通过计算欧式距离（Euclidean Distance）判断用户当前坐标与各个人格动物老家的远近，再利用反距离权重（Inverse Distance Weighting）推导得出精细的百分比结果 。
- **15维衍生光谱**：结合核心维度与模拟人性的混沌因子，额外推导出“自尊自信”等15个维度的五分制图谱，并配以通俗易懂的术语解释展示在前端 。
- **算法纠偏**：测试中发现全选某一项时概率计算失真，针对算法核心逻辑与题库分布进行了调整，以确保结果具有明显的区分度（主标签需占50%以上） 。

## 四、 视觉交互（UI/UX）与前端表现

- **动态视觉**：取消了纯黑背景，改为设计高级的“可动粒子（缓慢移动的星座图）”以呼应“棱镜”之名 。要求图鉴图片始终为彩色，并在鼠标悬停时伴随轻微发光效果 。
- **答题体验**：增添了随时跟随滚动页面的答题进度条，并支持修改已答题目 。
- **引力波辐射星图**：结果居中展示，其他相关标签按指数型距离（非线性）随机环绕，由连线标出距离 。后续优化了星图逻辑，去除了多余连线，改用“小爱心”和“排斥图标”直观展示最合得来与最合不来的人格，并缩减了星图冗余的外框范围 。
- **移动端适配**：调整了主图文案（“置身于4D引力波与15维人格光谱中...”）的排版，解决了手机端单行只有一个字的问题 。

## 五、 社交裂变：朋友圈专属小卡设计

- **潮玩小卡风格**：将普通分享海报升级为流行的潮玩小卡（prisma toy card）样式，并将生成入口提前至结果页最上方 。
- **智能取色与排版**：针对头像与文字重叠的UI问题，设计了智能取色方案（上方拼块取头像第一行居中像素颜色，下方拼块取最底行居中像素颜色），完美解决过渡留白问题 。
- **卡片细节**：重新布局了小卡内百分比文字的居中对齐问题，并在简介右侧植入了直接访问GitHub Page网页的二维码 。

## 六、 文档说明与隐私保障

- **代码分析与说明文档**：要求AI读取 question_v2.ts 和 Result.tsx 代码逻辑后，重新生成完善的项目 Readme 文件 。
- **安全确认**：明确询问并确认该发布在 GitHub Page 上的纯前端网页产品不会收集和泄露测试者的个人信息 。
